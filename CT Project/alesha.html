chapter 2: the coding arc

although i initially planned on coding in C# while using Unity to code my shoot 'em up game,
there were a couple of reasons of why i decided to use Godot instead:

  • my 4-year-old laptop would probably combust trying to load up Unity.
  • i've already had Godot installed before doing this case study.
  • Godot is completely free! unlike another game engine...

now, almost the entire time that i was coding my game, i mistakenly thought that i was coding in C#.

it wasn't until i actually read Godot's official coding guide that i found out that Godot has their OWN programming language, named GDScript.

here is an example of how different GDScript is from C#:

yeah... uh... pretty embarrasing mistake!
but i was halfway done with the game and i wasn't willing to switch all of my coding to C#,
since most of the coding tutorials that gave step-by-step instructions of the game that i was trying to make use GDScript anyway.

and besides, i think it would be interesting to work with a programming language that's exclusive to one game engine and compare it with the others.

here is a day-by-day account of the the kind of progress i did during the whole time i coded the game, and how i felt about it!
